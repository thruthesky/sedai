Getting Started with SEDAI

Install, initialize, and create your first SED project

1

Installation

SEDAI can be installed globally or used directly with npx. Both sedai and spec commands are available.

Global Installation

$ npm install -g sedai

Or Use with npx (No Installation Required)

$ npx sedai --help
$ npx spec --help
Note: Both sedai and spec commands work identically. Use whichever you prefer!
2

Initialize Your First Project

Use the spec init command to create a new SED project with initial specification files.

Interactive Mode

Run the command and answer the prompts:

$ npx spec init

Non-Interactive Mode

Provide all options directly:

$ npx spec init -n "my-app" -s "My awesome project" -a "Your Name" -e "your@email.com"
What happens: The command creates a ./specs directory with 4 specification files, each containing a YAML header with your project information.

4

Understanding Generated Files

After running spec init, you'll have these specification files:

The central index file that provides a detailed table of contents (DTOC) for all your specifications.

Contains:

  • Project overview and summary
  • Links to all specification documents
  • Brief descriptions of each spec file

Specification for database design, schema, and setup procedures.

Should include:

  • DBMS choice and version
  • Table structures with fields and types
  • Indexes and foreign keys
  • Migration procedures

Specification for backend architecture, APIs, and business logic.

Should include:

  • Technology stack and framework
  • API endpoints and routes
  • Authentication and authorization
  • Business logic and functions

Specification for frontend design, components, and user experience.

Should include:

  • UI framework and design system
  • Component specifications
  • User flows and navigation
  • Responsive design breakpoints
5

Next Steps

Now that you've initialized your project, here's what to do next:

  1. Complete your specifications - Fill in the generated templates with detailed requirements following SED principles
  2. Validate your specs - Use spec doctor to check completeness (when available)
  3. Calculate spec scores - Ensure your specifications score 90 or higher before development
  4. Instruct your AI - Provide specifications to AI and watch it implement exactly what you've defined
  5. Review and iterate - When changes are needed, update specs first, then re-implement
