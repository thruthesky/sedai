What is Spec-Exact Development?

Spec-Exact Development (SED) is an AI-driven development paradigm in which the AI never deviates by even a single line from the specification.

"If the spec is wrong, the product is wrong  by design."

Background

Created by Song Jaeho on November 4, 2025 to overcome the limitations of vibe coding and ensure consistent, predictable AI-driven development.

Problems with Prior Approaches

  • Specifications, context, and required skills are often described vaguely
  • MCPs provide only fragmented pieces of information
  • AI freely interprets requirements, creating unpredictable outcomes

SED's Solution

  • Provide AI with complete blueprints
  • AI follows the blueprints precisely without making inferences
  • Specifications become the absolute standard that guarantees consistency

Core Principles

SED is built on fundamental principles that ensure specification integrity and implementation accuracy

1. Spec-Exactness

Development implements only what the specification defines. If the specification is incomplete, the AI must immediately return a Spec Error and stop development.

2. Spec Completeness Scoring

Before development begins, the AI evaluates the specification and assigns a score from 0100. Development may start only when the score is 90 or higher.

3. Spec Is the Law

AI never attempts to infer human intent. Ambiguous sentences in the specification are ignored and not implemented. Guessing is forbidden; only written instructions are executed.

4. AI's Role in Validation

While AI cannot modify specifications, it analyzes logic for inconsistencies, executes tests, and provides advisory feedback to developers who maintain full control over specification content.

5. No Incremental Modifications

Iterative adjustments through progressive prompting are prohibited. When AI output is unsatisfactory, update the specification first, then instruct AI to re-implement based on the updated spec.

6. Complete Specification

Specifications must include class names, function names, implementation details, data structures, control flow, complete source code, code comments, CSS styling, user-facing text, and internationalization.

Quick Start

Get started with SEDAI in just a few minutes

Installation

# Global installation npm install -g sedai # Or use with npx (both commands work) npx sedai --help npx spec --help

Initialize a New SED Project

# Interactive mode (prompts for user input) npx spec init # Non-interactive mode (all options provided) npx spec init -n "my-project" -s "My awesome project" -a "Your Name" -e "your@email.com" # Partial options (remaining prompted interactively) npx spec init -n "my-project"

Available Commands

# Initialize new project (fully implemented) spec init [options] -n, --name <name> Project name -s, --summary <summary> Project summary/description -a, --author <author> Author name -e, --email <email> Author email # Validate specifications (coming soon) spec doctor [options] # Validate single file (coming soon) spec validate <file> # Calculate specification score (coming soon) spec score <file>

Development Workflow

A systematic approach to spec-exact development

1

Preparation Phase

Write comprehensive specification storylines (100+ items) and convert them into blueprint-level detail. Include database specs, feature specs, routing, functions, UI/UX requirements, testing specs, and deployment configs.

2

Validation Phase

AI analyzes and scores the specification (0-100). Development proceeds only when the score is 90 or higher. If insufficient, identify gaps and revise the specification.

3

Execution Phase

AI implements precisely what the specification dictatesnothing more, nothing less. Anything outside the specification is ignored. Code changes and spec updates are inseparable.

4

Verification Phase

Automated testing based on the specification. Test cases and expected outcomes are derived directly from the spec. Every feature is evaluated by: "Does it match the specification?"

5

Deployment Phase

Human involvement is mandatory. AI generates deployment scripts, environment configs, and CI/CD procedures, but humans review and execute the actual deployment commands.

6

Operation Phase

AI monitors, reports, and recommends automated fixes based on operational specs. When new requirements appear, follow the loop: update spec ’ re-evaluate score ’ redevelop.
